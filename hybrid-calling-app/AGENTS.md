# Agent Instructions for Hybrid Calling Project

## Branding

*   **Brand Name:** Hybrid Calling
*   **One Liner:** AI Agents That Talk Like You — 24/7. No Holidays. (Used in site metadata description)
*   **Tagline:** Your Voice. Your Intelligence. Your Closers.

## Development Notes

*   The project is built with Next.js (App Router), TypeScript, and Tailwind CSS.
*   Framer Motion is used for animations.
*   Supabase is the chosen backend for authentication and database.
*   Stripe for payments.
*   Twilio for communication APIs.

## Coding Conventions

*   Follow standard React/Next.js best practices.
*   Ensure components are well-structured and reusable.
*   Use descriptive variable and function names.
*   Comment complex logic.
*   Keep accessibility in mind.
*   All code changes should be tested if possible.

## Asset Placeholders

*   For features requiring specific visual assets (e.g., 3D avatars, complex animations, specific icons) that are not easily generated or found, use descriptive placeholders. Clearly mark these placeholders in the code or commit messages.
*   Example: `<!-- Placeholder: 3D Avatar -->` or `<div>[3D Avatar Placeholder]</div>`

## Premium UI Polish

*   Strive for a premium look and feel as per the project description (Stripe, Linear, Apple).
*   Implement animations subtly and effectively using Framer Motion.
*   Pay attention to details like spacing, typography, and color consistency.
*   Glassmorphism, neon glows (subtle), 3D card tilts, floating sparkles, and gradient fogs are desired effects. Apply them judiciously.

## Commits

*   Commit messages should be clear and follow conventional commit standards if possible (e.g., `feat: Implement hero section UI`).
*   Reference issue numbers if applicable (though not used in this current context).
